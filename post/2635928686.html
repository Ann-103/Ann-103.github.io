Chapter 0 概述和发展历史

对于网络中数据的传输，可以分为接入链路，寻址路由，传输控制，网络安全等方面

计算机网络的一些基本概念

一些专业名词定义

交换机（switches）：实现线路复用，为两个非直接相连的设备提供数据转发

链路（link）：连接两个或者多个节点的物理介质，分为点对点链路和多路访问（共享）链路

路由（连接不同网络的交换机）：如相隔多个网络的两个通信节点找到数据转发路径，执行路由操作的交换机通常被称为路由器Router。

操作系统：对节点上的资源进行管理，为软件提供运行环境

网络源于分布式系统-> 网路操作系统（DNOS），云网融合由传输功能到计算存储网络资源等功能靠近。例如IOS。

协议：定义了节点间的通信规则密码，包括信息格式、交互流程、解释处理

DNS：域名系统（Domain Name System）

ARPAnet: 阿帕网，Internet前身（Advanced Research Projects Agency network）（军事）

NSFnet:美国国家科学基金会网络( National Science Foudation network)(科研背景)

ISP（Internet Service Provider）的网络互连而成IPv6标准。用户只要运行TCP/IP协议，可以通过ISP连接到Internet（还要给钱）（商业）

概述

因特网，万维网，局域网，WiFi，电话线，早期的网络构建到底是什么

  • 1946 第一台ENIAC

  • 远程联机控制系统：所有终端都通过电线连接到CPU和磁盘驱动等硬件设备

有意思的是现代由于CPU利用率不到20%所以很多公司并不配备单独的电脑而是大家只用终端无线连接一个CPU

  • 由于电脑终端们到资源（打印机），直接连接太远太多，所以考虑将所有终端都连接到交换机上，交换机单独连接到资源上，这个线路会给不同终端错开复用

  • 交换机（switches）：实现线路复用，为两个非直接相连的设备提供数据转发

    • 控制处理器：管理交换，执行高层协议

    • 交换结构：将流量到处到输出关口

    • 输入输出端口：传递和接受包

路由：如相隔多个网络的两个通信节点找到数据转发路径，执行路由操作的交换机通常被称为路由器Router。

路由,“连接不同网络的交换机”

路由过程（Routing Process） :路由器中执行相同的协议OSPF/BGP -> 交换拓扑信息 -> RIB -> 如何进行数据的转发 -> 目的地

RIB,“交换路由信息表”

  • 所有的RIB中存储着路线，用来描述路由过程
  • 虚线表示出入口和再分配的路线
  • 接口进行简单标号
  • 实线是接口之间的物理连接

硬件

  • 计算机，主机Host：用户终端和服务器
  • 网络设备：交换机和路由器
  • 所有的机器都是节点Node
  • 链路
    • 点对点（Point-to-Point）链路
    • 多路访问（Multiple Acess） /共享链路

软件：

  • 操作系统：对节点上的资源进行管理，为软件提供运行环境
  • 网络操作系统 ：IOS（cisco）
  • 协议：信息格式，交互流程，解释处理

计算机网络：一系列计算机、链路和网络设备的集合。

  • 物理上：链路和网络设备将计算机连接起来确保了拓扑（图）上的连通性（任何时刻的任何节点之间都有数据传输的路径）

  • 功能上：提供了数据传输功能（协议）。并且可以实现信息服务（即时通信等）和资源共享（打印、云计算等）。

通用性：计算机网络可以在计算机上运行软件实现各种网络应用，也可传输文件视频等多种数据类型。将传统的电视网络、有线电话网络等通过不同协议统一。

互连网（internet）：一些独立的计算机网络互相连接。通过网络嵌套可以构成任意规模的互联网，在此简称网络。这些网络之间的通信协议可以是任意的。

因特网\互联网（Internet）：各种类型网络互连而成，均运行TCP/IP协议族。是世界上最大的互连网。

IVP6 发展史

IVP6 发展史

IPv4(Internet Protocol version 4)：互联网通信协议第四版，是一种无连接的协议，操作在使用分组交换的链路层（如以太网）上。此协议会尽最大努力交付数据包，意即它不保证任何数据包均能送达目的地，也不保证所有数据包均按照正确的顺序无重复地到达。这些方面是由上层的传输协议（如传输控制协议）处理的。

IPv4 地址危机，四亿地址分配给节点。IANA（International assigned number authority）,将ip地址划分给地区的 ip，APNIC：亚太地区，进一步分给国家isp。

IPv6(Internet Protocol version 6)/IPng：下一代互联网协议/ 国际协议第6版，为了解决IPv4地址枯竭问题

chapter 1 网络的体系结构和参考模型

网络体系结构：层和协议（功能）的集合，层表示了网络协议（功能）的组织结构。

网络体系结构用于指导网络设计，比如定义协议的集合和协议之间的组织结构，大部分计算机网络包括因特网均使用分层的网络体系结构。

分层优点：将建造一个网络的问题分解为多个可处理的部分，每一层解决一部分问题。并且这种模块化的设计可以在不影响别的部分功能的前提下进行修改或者添加新的服务、

分层的一个例子

模块设计：格式，交互，解释

  • 文件传送：与文件处理相关（确信对方做好接收和存储文件的准备以及协调好一致的文件格式）
  • 传输控制：与数据传输有关，和失败重传的控制有关。
  • 网络接入：确保传输的数据满足网络的要求（比如传输的帧格式，帧的最大长度等）

B PDU/PCI/SDU的相互关系以及所谓的封装思想

协议：协议定义了通信双方所使用的数据结构和数据处理、交互流程

  • 封装（发送向下到物理层）：在发送节点上，每一层都给来自上层的协议数据单元（PDU）加上代表协议控制信息的头标或者尾部，向下层协议递交
  • 解封装（接收向上到应用层）：在目的节点上，每一层协议根据该层协议控制信息完成该层PDU处理后，去掉该层协议相关的头标或者尾部，向上层协议递交

A ISO-OSI模型

OSI

OSI体系结构由ISO于1983年给出，基于OSI参考模型 即开放式系统互联参考模型

路由器：只执行物理层、数据链路层、网络层的功能，可能不止这几层。

操作系统控制，其中前三个在网络路由器上实现

  • 物理层：（PDU：Bit） 缆线、信号的编码，网络接插件的电、机械接口

    • 激活物理连接发送数据，准备必要资源（缓存区等）

    • 原始比特流（同步、异步模式 ；全双工、半双工传送模式）

    • 相关协议：接口物理规范；设备间信号的传送方式（调制、编码、信号处理和传输线路的特性匹配）

    • 设备与传输介质：信号放大器、集线器；传输介质为电缆、光纤、无线电波

  • 数据链路层（PDU：帧） 成帧，差错控制、流量控制，物理寻址，媒体访问控制

    • 发送方成帧和接收方帧定界（比特流中帧的开始结束位置）、帧破坏、丢失和重复所出现的问题；流量控制：如何避免一个快速的发送方淹没掉一个慢速的接收方（接收方能力不足）、广播式网络（共享链路）中的共享信道的访问控制

    • 相关协议：高级数据链路控制规程 HLDCP（High Level Data Control Procedure）；点对点协议PPP；CSMA/CD和CSMA/AD（ Carrier Sense Multiple Access with Collision Detection/Avoidance ）

    • 设备与传输介质：网桥（Bridge）,L2交换机（Switch），无线接入点（AP）

    帧：将bite进行划分

    时分复用的实现：共享信道访问控制，使得上网有序

    网桥：历史遗留，现在基本上都是以太网的交换机。AP就是一种网桥。

  • 网络层（PDU：分组）路由、转发，拥塞控制

    • 网络互连的问题；路由：根据数据目的地址确定目的网络的最佳路径；拥塞控制（拥塞主要发生在路由器上）

    • 相关协议：ATM（发送数据前连接、数据单元为固定长度信元，携带连接表示并在连接中投递）；IP（不建立连接，数据单元为 分组，可以沿着不同路径到达目的主机）

      ATM：定义了从物理层到网络层所有的技术，理论上可以单独组网，固定长度信源（cell 53 字节）

      IP：不定长度分组

    • 主要设备：路由器/L3交换机

  • 传输层：为会话层提供与下面网络无关的可靠消息传送机制

    • 向上层提供不同类型的传输服务、端到端：在主机上运行，非路由器，流量控制（和端到端的服务有关，如果可靠传输需要有流量控制，接收端能力不够）
    • 传输层服务
      • 面向连接：数据传输之前连接两个端点协商参数
      • 无连接：直接发送数据
    • 传输层以上协议的设备通称 网关（Gateway）

    路由器仅进行到网络层

  • 会话层： 负责建立（或清除）在两个通信的表示层之间的通信通道（会话，比如一次通话），包括对话控制、交互管理、同步，异常报告、令牌管理、禁止同时执行关键操作等。
  • 表示层：在两个应用层之间的传输过程中负责数据的表示语法比如文本图片。以及加密和压缩。
  • 应用进程（层）：定义满足各种应用需求的协议，处理应用进程之间所发送和接收的数据中包含的信息内容。
  • 设备均称：网关（Gateway）

A TCP/IP模型

TCP/IP参考模型（Internet 中的两个核心协议）

只要运行这两个协议（主要是ip），则可以进行网络互连。

TCP/IP 协议栈

  • 下层：网络接入技术，有线无线电缆等
  • 上层：网络应用

主机至网络层（或网络接口层、链路层），在 TCP/IP 模型中很少提及，指链路必须完成什么功能才能满足无连接的互联网络层的需求。

  • 链路（link）：一个广播域，和OSI不同（OSI中链路指物理连接或信道例如一个点对点的电话线）
  • 广播域：任意一个节点可以在数据链路层通过广播的方式到达任意一个节点。 广播域可以被部署在同一个局域网或者被桥接到其他的局域网。
  • 一跳：跨越一个广播域，同样指的是路由概念

广播域对应的是物理层和数据链路层。发布一个帧，通过该层连接的主机设备均可以接受并处理。

二级交换机连接的网路中所有的节点在同一个链路上。链路的边界是三层设备：路由器或三层交换机。

互联网层：（网络体系结构贯穿关键层，大致对应网络层）：将分组投递到最终的目的地 ，提供无连接（想法就发）和不可靠（数据包丢了就丢了）的传输。是一种尽力服务模型

提供非连接的分组交换功能：主机将数据包注入到任何网络并让这些数据包独立的到达接收方（可能顺序也不同），这里指的是一般意义上的internet。

协议：IP：定义官方格式； 辅助协议：因特网控制报文协议（ICMP）、、

尽力服务模型保证了Internet的可扩展性，复杂的网络功能由用户终端设备来实现，而网络只负责分组的投递。

  • 也称为IP层，主要是IP协议

传输层：提供可靠的面向连接的传输层协议 TCP 和不可靠的非连接传输层协议 UDP（很少使用）。

应用层：向用户提供应用服务，简单包含了会话层和表示层的功能。

  • 域名系统DNS：域名 <–>IP地址
  • FTP协议：文件传输应用
  • Telnet协议：远程登录应用
  • SMTP协议：发送电子邮件
  • HTTP协议：WWW应用

域名：将人使用的域名解析到机器使用的IP地址

应用层协议：交互流程和规范

应用层程序：需要实现协议并提供图形化界面等其他功能

本课程采用的是五个部分的模型，物理、数据链路、网络（前面三部分包括物理网络、以太网、无线网、物理网络、IP互联/路由等）、传输、应用（后面两部分包括具体的应用 ，数据库，DNS,SIP等）

现代浏览器使用HTTP或者HTTPS的协议，对应传输层是TCP协议

OSI 与 TCP/IP 模型比较

相同点：

  • 都是基于独立的协议栈概念

  • 两者都有功能相似的应用层、传输层、网络层

不同点：

  • 在 OSI 模型中，严格地定义了服务、接口、协议；在 TCP/IP 模型中，并没有严格区分服务、接口与协议

  • OSI 模型支持非连接和面向连接的网络层通信，但在传输层只支持面向连接的通信；

  • TCP/IP 模型只支持非连接的网络层通信，但在传输层有支持非连接和面向连接的两种协议可供用户选择
  • TCP/IP 模型中不区分、甚至不提起物理层和数据链路

仅定义TCP/IP协议使得网络变得简洁而有丰富的可扩展性。

Internet社区：主要有IAB（体系结构委员会）、IETF（工程化和开发力量）、IESG（前面那个的技术管理和标准化）等，开放的（free），RFC

IETF：因特网工程任务组（The Internet Engineering Task Force）

IEEE：电气电子工程师协会，电气工程、通信和计算机领域中的标准（无盈利的学术组织），主要做数据链路层以下的标准，802标准，主要是局域网。

ISO：国际标准化组织学科国际标准，受ANS影响较大

协议一般分为标准协议和私有协议，一般app使用的都是标准协议。

网络性能度量（延迟）

速率/数据率/比特率：bps（b/p）;kbps(kb/s); Mbps(Mb/s);

网络容量（速率） 度量时

  • 1kbps=103^33bps,1Mbps=103^33kbps,1Gbps=10^3^Mbps,1Tbps=10^3^Gbps

数据量（存储容量）度量时

  • 1kB=210^{10}10B, 1MB=210^{10}10KB, 1GB=210^{10}10M, 1TB=210^{10}10G

带宽（Bandwidth)

  • 信号具有的频带宽度，单位是Hz（赫兹）
  • 数字信道或者网络所能传送的**“最大数据率**”，即信道或者网络的实际容量，单位是bps

网络吞吐量（Throughput）指网络的可用带宽，也就是应用感受到的有用带宽

由于网络协议栈的每一层协议都有相应的头标和尾部等开销，还有协议实现机制的开销，因此准确的吞吐量应该指明是哪一个协议的吞吐量

延迟（Latency/Delay）

名称 位置 描述 公式
发送延迟（Transmit）， 发送器 与发送分组的大小和带宽有关 Transmit = Size / Bandwidth
传播延迟（Propagation） 链路 与距离和电磁波传播速度有关 Propagation = Distance / Speed（一般都按照光速）
处理延迟（Process） 节点A 节点对分组进行处理导致的延迟例如提取和分析分组的头标、校验、查找路由等（相对固定） （不考虑）
排队延迟 （Queue） 节点A 中间路由器或者交换机将分组转发出去之前将它们存储的时间（最复杂） -

延迟抖动（delay jitter）：端到端延迟变化，主要由中间路由器上排队延迟的不确定性引起

延迟和带宽乘积：

  • 单向延迟× 带宽: 第一个比特到达接收端之前，发送者能发送的比特数

  • 往返延迟（RTT） × 带宽: 发送端在接收到接收端给出的确认之前能够发送的比特数
    链路的延迟带宽积又称为以比特为单位的链路长度

一般计算2*传播延迟+发送延迟
