Windows科研环境下的电脑配置与软件推荐

发表于2024-09-23|更新于2024-11-15|Tools
|字数总计:3.4k|阅读时长:11分钟

包括美化以及效率工具在内的电脑配置安装，会推荐一些适合科研的软件。

通用效率工具

Utools+everything

下载：

  • utools：效率管理
  • everything：快速查询

简单来说，utools就是一个万能的接口工具箱，完全可以通过utools启动电脑中任何软件。它也内置了一些插件工具，可以通过插件市场下载使用。此外，utools结合everything可以实现电脑文件的超快速检索。

分别在官网下载安装后，Alt+space呼出utools工具栏，工具栏里可以启动软件或者搜索文件等，更多功能可以去官网查阅。

推荐在utools设置里开启开机自启动，并把快捷方式固定在电脑任务栏，同时在everything里也设置一下开机自启动。

完成后，不依赖桌面图标开启软件，可以替代电脑原来的软件快捷方式、搜索功能、文件查找功能等。

everything的一些简单的使用技巧：

Geek

下载：Geek官网

免费卸载软件的软件，可以检测卸载残留，注册表相关信息，卸载得比较干净。大部分软件都可以通过Geek卸载，如果不能，考虑杀毒吧。

那些捆绑下载的软件，小×壁纸，鲁×师，3×0杀毒等等，该卸载的就卸了，没有弹窗广告和流氓软件自动下载的电脑才是好电脑。

Typora

下载：Typora 官方

非常好用的MD编辑器，即时渲染，官网付费，也可以搜索之前的免费版本下载。

另外如果实在不想用typora，Utools里的MD编辑器也不错，简约快速，支持导出，唯一缺点是Utools目前的备份是Vip功能。

Picgo（可选）

下载：GitHub - Molunerfinn/PicGo:

图床工具，简单来说就是将图片上传到服务器（图床）上，通过URL链接进行线上访问的工具，写博客的时候很好用。

WSL和终端设置

下载：Microsoft商店

所有的shell都可以集成在终端中，结合utools呼出方便，而且更加美观，也方便复制粘贴。

火绒

下载：火绒安全

防火墙、广告拦截、电脑保护。国产个人免费，不会有捆绑下载或者广告或者卸载不了的情况。基本上电脑下载这一个杀毒软件就可以了。

另外推荐一下NOD32杀毒软件，如果对安全性要求较高的话可以考虑下载。

代码环境

Visual Studio

下载：Visual Studio Community Editio

按需安装对应组件即可，基本上可以满足大部分的代码需求。

VScode

下载：官方

轻量级，强大而且很好看，但是需要自行配置一些文件，运行的时候可能会出一些奇怪的BUG。

安装过程中勾选添加到PATH即不用手动配置环境变量。

简单配置：

  1. 插件搜索安装c/c++（前三个），code runner。
  2. File->Preference->Settings->extension，将Run Code Configuration-Run In Terminal、Save File Before Run前面的复选框选中。
  3. Windows下载安装编译器MinGW-w64 - for 32 and 64 bit Windows download | SourceForge.net，然后将bin写入环境变量。
  4. Terminal-Configure Tasks…-，选择C/C++:gcc.exe生成活动文件（有时候需要重启一下电脑），会自动生成一个.vscode的子文件夹，并自动创建一个tasks.json的文件，不用做任何改动。

VMware

下载：Download VMware Workstation Pro

下载路径更改，其他默认安装即可，创建虚拟机基本上就是个独立的linux系统了，记得关机的时候从工具栏关闭，不要直接叉掉，然后保存快照是个好习惯。

下载Ubuntu ：下载Ubuntu桌面系统 | Ubuntu

创建对应虚拟机，在终端中安装所需环境，可以安装vim作为编辑器。

WSL2

更加轻量化的linux环境，WSL和windows会共用同一文件系统。

其实更推荐使用WSL进行简单的科研实验，不要把时间浪费在配置虚拟机上

  3. 打开 Microsoft Store，并选择你偏好的 Linux 分发版下载即可

菜单点击Ubuntu即可运行，也可以集成到windows terminal中

Git

下载：官网

多人合作代码工具，结合Github等托管平台使用。下载后会自动添加环境变量，Git Bash 会作为应用程序添加进系统，有关Git的命令我们在这个窗口进行（右键git bash here即可）。

安装过程中可能会有一个默认”Use Git from the Windows Command Prompt“的选项，表示命令在window的cmd命令行中也可以运行。

进行电脑设置：

  • 在cmd设置姓名和邮箱地址，这里的信息对应了电脑提交的署名

  • 命令生成~/.gitconfig文件，可以设置

    从而让命令的输出有更高可读性

建立电脑与你的github账户的连接，这样电脑就有权限向你的github账户推送：

  • 生成ssh key，解决本地和服务器的链接问题，首先打开电脑文件夹，找到C:\Users\名字.ssh文件夹并删除，git bash:

    输入三次回车

    打开.ssh文件夹，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容

  • 打开github主页，进入setting-> access->ssh and gpg keys，把复制的内容粘贴进去，自己命名保存

    电脑端输入ssh -T git@github.com验证，如果崩出来个选项输入yes回车，如果返回Hi！ ，You’ve…即成功，你可以往你自己的账号推送代码了。

桌面

桌面配置以简洁美观为主，下面是配置后的效果：

PS：windows更新后，底下任务栏也自动居中了。

TranslucentTB

任务栏透明插件，直接在微软商店下载即可。

Nexus

美化任务栏的软件，可以在网上搜索下载，之后需要进行配置，这里可以按照个人喜好设置。

Nexus暂时没找到官网，如果找不到资源的话bilibili上有up主分享

Nexus下载后右键任务栏打开首选项:

  1. 选择：位置->停靠在底部，设置停靠偏移屏幕的边缘像素
  2. 选择：外观->Dock透明度，设置透明度为零，这里Dock大小也可以按照个人喜好更改，取消勾选“鼠标经过时显示图标标签”
  3. 选择：效果->自己选择喜欢的效果
  4. 选择：声音->禁用所有语音、禁用所有声音
  5. 选择：行为->锁定图标并防止它们被拖动、防止Dock被鼠标指针拖动
  6. 选择：一般->隐藏系统任务栏
  7. 点击应用

删除原有的插件：右键初始的图标信息，点击从Dock中删除。

新建文件夹，将桌面上想要显示的图标快捷方式复制进去，然后右键Dock新建项目，点击查找项目，分别添加上面的快捷方式。

代码环境配置

python

一些介绍：

Anaconda：可以便捷获取包且对包能够进行管理，同时对环境可以统一管理，包含了conda、Python、numpy在内的超过180个科学包及其依赖项。

conda：包及其依赖项和环境的管理工具， 不会影响系统自带python。

pip：用于安装和管理软件包的包管理器。pip可以允许你在任何环境中安装python包，而conda允许你在conda环境中安装任何语言包。

重装Anaconda

官网下载即可。安装过程中不勾选add anaconda to the system path 则仅在运行才会将anaconda安装的py添加到环境变量，不影响电脑原本的py，在cmd中查看py版本仍旧是原本的。

但是我安装的最新版本好像并不是这个界面了，直接默认安装

  • 添加环境变量。utools搜索系统环境变量，然后点击系统环境变量，在path中添加如下路径（如果默认添加过环境变量则无需再次添加）：

  • cmd查看安装版本

  • cmd更改镜像源

  • cmd中使用conda环境管理

  • conda 包管理

    conda install xxx：这种方式安装的库都会放在anaconda3/pkgs目录下。好处：conda不同环境中需要库可以直接复制，不会重复安装

    pip install xxx：如果当前conda环境的python是conda安装的，和系统的不一样，那么xxx会被安装到anaconda3/envs/current_env/lib/python3.x/site-packages文件夹中，如果当前conda环境用的是系统的python，被安装到~/.local/lib/python3.x/site-packages文件夹中

  • conda激活环境后，可以直接运行py文件或者使用自带的ide运行简单的文件

  • 不同位的python

Pycharm

官网安装即可，每创建一个项目都需要指定相应的Python解释器

  • 创建的时候选择环境
  • 修改：file-setting-Project Interpretor-齿轮-conda environment-existing environment-浏览安装包中env的对应文件夹下的环境

内置jupyter notebook的网页交互：new新页面就可以实时交互了
