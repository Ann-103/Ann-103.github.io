c 语言简介

C语言简洁灵活，可移植性强，执行效率高，允许直接访问物理地址（慎重），能执行位操作等。可以说是比较接近机器语言的一门高级语言，同时有汇编语言和高级语言的优势。

编译和链接

程序员编写的代码首先是以文本文件形式存在，扩展名为.c ，为了得到可以直接执行的机器语言（.exe文件），需要由编译器把源代码转化为中间代码（比如目标文件代码 .obj 等），然后由链接器把中间代码和其他代码合成可执行文件（.exe）。

值得说明的是，中间代码（.obj）相比最终目标（.exe）缺少启动代码和库函数或者自己编写的头文件函数的链接。链接器起到的作用就是把你的代码中引用的函数找到它的定义出处，也就是库或者你自己写的头文件。同时，这也是c 模块化特点的体现。

另外在自己编写头文件的时候需要注意的一点是不能重复编译，比如vs会默认在你自己编写的头文件中加一行 #pragma once 。

c的编译可以通过操作系统使用c编译器完成，比如linux会有GCC的安装提示，编译的使用gcc inform.c命令，windows软件安装包并没有c编译器，需要下载Cygwin和MinGW，然后通过命令行的方式使用GCC编译器。

GCC是GUN（一个开发大量免费Unix软件的集合）项目的一个产品，或称为GUN编译器集合，可以适应诸多硬件平台和操作系统，许多使用gcc的系统使用cc作为gcc的别名。（但cc是一个过时的命令）

实际编程中，大部分程序员使用的是IDE，IDE可以看成一个集成开发环境，比如常用的VScode 、visual studio 等，利用IDE可以快速开发C程序，把之前的编译链接过程等简化，并且可以debug，还有其他的功能的插件。

有的学校会用dev c++作为基础学习ide，我们自己可以尝试实用vscode或者visual studio这种更符合实际开发环境的ide

数据类型

c语言需要提前说明变量类型来分配存储空间及确定可支持的操作。

  • 基本类型: 整型、实型、字符型；
  • 构造类型有: 数组、结构、枚举、共用体；另外还有指针类型和空类型。

整型数据

int （%d十进制表示；%o八进制表示；%0x十六进制表示）；

short（%hd）；long（%ld）；unsigned （%u）；unsigned short；unsigned long 。

实型数据

float（四字节，单精度型，七位有效数字）；double（八字节，双精度，15到16位）

表示形式：十进制小数（3.14），指数3.0e+5

指数形式表示规则：小数和整数不同时不存在。中间不能加空格。.2e4 .2 100. 都是合法的。

字符型数据

数据类型转换

运算中一般整型转化为int，实型转化为double，后者优先。可强制转化，eg：(float)5/2=2.5 ！= 2 (float)(5/2)= 2。

另外一些数据的强制转换需要考虑符号位，是否要进行符号扩展以及是否会溢出等(数字大到一定程度反而成为负数了)。

且一般浮点数转化为整数会舍弃小数，具体的还要看编译器，自己试一下。

函数

函数之间相互独立，可以相互调用，自己调用自己即递归。

形参：函数在定义的时候参数是形参，只有在运行函数的时候会分配空间，运行完释放。

实参：是调用函数的时候，实参对形参的值的传递是单向的，一些改变原本实参的函数不能通过简单的调用实现，需要依靠指针，或者数组的传递本身也是地址传递，会影响到实参。具体的在指针部分介绍。

指针的作用之一就在这里，如果单纯传递某个变量的值的话，那么操作对象就是这个值，和变量无关，但是如果你把变量存储的地址发送过去了，那么操作对象就可以直接变成变量的老家，相当于添加了个中间商。

数组就是传递了起始数据的地址比如int max(int num[],int n)//n表示数组大小，如果是多维数组的传递的话，必须要把列数说明，比如 int max(int num[ ][4])

return expression; 返回对应数据，如果void则不返回，但是可以通过 return ;语句提前结束函数

数组和字符串

  • 数组是同一数据类型的集合，数组定义的时候必须要输入具体多少个数据和这些数据的类型，在c语言中不可以定义动态数组，即数组中元素的个数是常量。
  • 在内存中数组是偏移量（下标）为0开始的一系列存储单元，要保证调用的时候不能越界。
  • 数据处理的对象只能是一个元素，不能是一整个数组（初始化除外）。
  • 多维数组可以看成一个递归的定义，比如二维数组是一维数组的数组，三维数组是二维数组的数组。在物理存储位置中，c语言是从低地址到高地址按行连续存储，先存第一行即 a[0][0],a[0][1],a[0][2],a[1][0]...

指针

指针是c的很重要的一部分，可以看作一个存储地址的变量。指针的存在使得代码难度上升，更容易崩电脑，也让代码紧凑高效。由于它自己就是个地址，也可以看成一个箭头，箭头指向的是地址对应的变量。

需要注意的是，在定义指针变量的时候分配的只有一个地址的空间，但是并没有对指向的数据进行分配空间，这一点和数组是完全不同的，数组是建房存数据的地方，指针只是一个箭头用于指向某块地址，所以在后面可能会介绍malloc函数或者new函数，这些都是为数组指向的数据分配空间的函数。

指针和数组之间的关系：

数组名表示的是数组首地址，不能修改，故数组的访问也可以通过指针实现，但一般能用下标解决就不用那么麻烦了

作为函数参数传递的时候指针和数组名是等价的，因为本质上传递的都是地址

链表：数据结构部分会有详细介绍，本文就简单结合后续小游戏里的示例进行说明，可以先简单理解为链表就是多个节点的集合，每个节点由变量值和指针组成，通过指针找到下一个节点的存储位置。

需要注意的是，链接和断链的过程要有顺序，否则会找不到原来的链接关系。

先简单来说一下链表的建立过程，分为有无头节点，头插法（倒序）和尾插法（正序）。

头节点：头节点不存储具体数值，起一个定位作用

  • 有头节点：初始化head指针指向的空间，头节点的下一个节点是null
  • 无头节点：直接初始化head指针为null

头插法：

  • 有头节点：插在头节点的下一个；先更改插入节点的指针指向原本的第二个节点，再更改头节点的指针head指向目标
  • 无头节点：插在最前面；先更改插入节点的指针指向原本的第一个节点，然后直接将head指向插入的节点

尾插法：和头插法相比，尾插法更为简单也更普遍应用。因为是直接插到链表最后,保证节点指向NULL,然后只需要尾节点指向新插入的节点即可。

例题： 验证哥德巴赫猜想，任一个充分大的偶数，可以用两个素数之和表示，编写一个c程序使得输入一个偶数，发现两个素数，其和等于所输入的偶数。

例题：乒乓球两队比赛，各出两人，甲队为A、B、C三人，乙队为X、Y、Z三人，已抽签决定比赛名单，A说他不和X比，C说他不和X、Z比，请变成找出三对赛手的名单。

例题：筛数法求素数，基本思想是将某个范围内的自然数从小到大排序，筛出非素数1，然后从剩下的数中选择最小的作为素数并去掉他的背书，重复直到剩下的数为空。

例题：幻方问题，将n阶矩阵中的数 1到n2n^2n2进行重新排列使得行、列、斜对角的数字之和相同。

这里有一个万能的解法公式，将1填在第一行中间后剩下的数依次在右上方填充，遇到边界就跳转到最后或者最左，已经填充则放在正下方。这样直观上可以保证，每一轮行上的数递增，然后起始的最小数字的行数分别是第一行，第三行，第五行，如果是五阶的话再到第二行，第四行开始的轮次递增，这样遍历可以使得每一行数的总和固定不变，同样从列的角度去看也是。而在对角线中，左下到右上的对角线是一个最中间的轮次，右下到左上则分别是不同起始行轮次的中间数字，可保证和不变。

为什么填充正下方的数字的时候不会遇到刚好需要填充的下一行不存在的情况，比如五阶的起始行分别是1、3、5、2，4，那么分别会在第2，4，1，3，5的地方需要向下填充，很明显最后到第五行需要向下填充的时候已经没有数字了。

例题：文件读写，写入五个同学的学生记录，分别为姓名，学号，成绩，写入后随机读取第三条记录并利用第二条记录替换。

有关于文件读写可能和不同编译器也有关系，比如vs可能会需要使用安全读写函数。另外在数据结构部分可能也会需要文件处理函数。例题就简单介绍到这里，另外有关指针的部分没有题目，可以看下一部分。

c语言结课小游戏

通过贪吃蛇的c语言小游戏来结束c语言复习的内容。

框架介绍

首先是主函数部分，这种c语言的小游戏一般会使用while死循环的方法不断刷新页面，然后期间更新部分内容的坐标。自己写一个小游戏可能更好的理解模块化编程的意义，这里通过链表表示蛇，从而不限制具体的长度。

主程序最好可以直观的看到这个游戏程序的结构，然后我们根据需要分别编写函数，以及确定好游戏需要的变量。

通用函数

分别介绍函数的功能,首先是全局都通用的一个函数，将标准输出的位置放在xy坐标，另外这里所有的x坐标是指横向，y为纵向

初始化欢迎页面和造墙，这里默认的是x方向要两格表示，不然会有莫名其妙的bug。

初始化和食物生成

然后是蛇的初始化和食物的生成函数，其中蛇是通过链表中的节点存储每一节的位置，食物的初始化需要保证在墙内且不能和蛇身重合。

关于链表的操作是一个难点，本游戏涉及的链表操作只有链表的建立（头插法），链表的遍历和增加节点，虽然用数组也可以实现相关功能，但是链表的灵活性要更好。

顺便说一下最后的善后工作，即链表的释放，使用free函数，参数是指针，表示释放指针所指地址位置的空间。
