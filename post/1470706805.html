博客搭建教程（一）

发表于2024-08-08|更新于2024-10-28

Hexo+Github page博客搭建教程

Windows上从零开始部署和美化个人静态网页，部分内容偏原理解释，方便出错后排查原因。

Hexo+Github的部署方式简单，网页架构更为轻量化且完全免费，比较适用于非专业人员搭建个人网站使用，本教程偏向于个人搭建记录。

PS：在本文的命令行中，所有的[]表示可选内容；{}表示在选项中选择其一，用|或者||隔开；<>表示必填的占位符，...表示参数可重复；

PPS：环境变量可以直接电脑左下角搜索，或则点击菜单->设置->系统->关于->高级系统设置->环境变量；

背景简介

Github page

Github pages平台是github提供的静态网页寄存服务，直接从Github上获取HTML、CSS和JS文件，构建过程运行文件发布网站。并不支持PHP、Ruby等服务器端代码，其他用户通过访问github.io的子域名或者绑定第三方域名就可以访问网页。

简单来说就是不需要自己买服务器存放网页源文件了，寄存在github上就可以，作为静态网页，即没有服务端数据交互的网页搭建很方便。

服务器：由处理器、硬盘、内存等构成，提供计算服务（文件、数据库、web）的设备，可以简单看成提供高可靠服务的特殊电脑。通过IP或者域名服务可以访问服务器，例如浏览器输入www.baidu.com可以访问百度的服务器。

Hexo

静态网页中用户仅可浏览内容，无法交互。前端使用HTML（框架结构）+CSS（颜色样式）+JavaScript（点击变色等行为）搭建，如果想要个性化网页搭建需要熟悉这些前端内容的开发。

Hexo使用Markdown解析文章并生成静态网页，是一个快速简洁高效的博客框架。安装前提有两个，分别对应后续安装的step1和step3。

  • Node.js(Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本)
  • Git

快速开始

Step 1：NODE安装

介绍：Node.js是一个可以运行JavaScript的运行环境，或者说是JavaScript运行时（Runtime），包括语言引擎和核心代码库等。将Node.js下载到本地后，本地运行js文件可以实现服务器后端服务。

在Node.js之前，JavaScript只能运行在浏览器中，引入后可以直接在计算机上使用。

方式一：直接下载NODE（不推荐）

如果你是电脑小白，只想运行这一个node项目，不许需要切换版本，那么直接在官网下载：Node官网，选择LTS（长期支持版本），默认操作即可。安装时会自动配置环境变量并附带npm，无需手动添加。

NPM是附带的node包管理工具，用户可以在NPM服务器下载所需要的包或上传包程序。

打开cmd输入以下命令以查看相关版本和更新：

cmd：windows（窗口键）+R，输入“cmd”即可打开windows命令行界面。

方式二：NVM（推荐）

NVM是Node.js的版本管理器，使用bash实现，非常有名。在安装之前，如果已经通过别的方式下载过node，则需要手动卸载。

NVM是在linux下用于管理node包版本的工具，后续衍生出了windows版本的nvm-windows。

NPM只能管理包的依赖，NPM在v5.2.0新增了npx命令，可以快捷的运用包中的命令行工具和其他可执行文件，让项目内部安装的模块用起来更方便。使用NVM可以管理多个NODE版本（包括npm和npx）。

彻底卸载之前安装过的NODE

  1. 卸载程序node.js，这里推荐使用软件geek（免费）
  2. 如果在安装目录中找到安装文件夹nodejs则删除，比如下载时的默认路径可能是C:\Program Files\nodejs
  3. 找到npm包的全局安装目录，一般是C:\Users\{User}\AppData\Roaming\npm并删除
  4. 检查path环境变量，确保没有nodejs或者npm的相关引用
  5. 重启电脑

安装nvm-windows

  1. 前往官方版本：https://github.com/coreybutler/nvm-windows/releases，下载nvm-setup.zip

  2. 选择无中文无空格的路径新建一个nvm和node文件夹，如果选择默认安装路径则一般安装在c盘的program file中，直接点击setup文件安装即可。

  3. 检查并安装不同版本node。打开cmd输入以下命令并查看：

linux上直接按照官方文档下载即可：nvm官方

环境变量

环境变量会自动配置，这里会添加两个环境变量NVM_HOME（安装目录）和NVM_SYMLINK（指向当前版本的软链接地址，比如node文件夹），然后在PATH中引用%NVM_HOME%和%NVM_SYMLINK%，出了问题可以自行检查是否链接成功。

卸载nvm

如果使用nvm体验不好，比如对prefix支持不好，启动慢，很多地方不兼容等。可以考虑卸载nvm，使用nvs。

  1. 列出所有已安装的节点包并卸载

  2. 点开下载nvm的文件夹，运行unins***.exe，运行卸载即可。

方式三：NVS（推荐）

NVS是跨平台的，基于Node编写，同样如果已经使用前面两种方式的话需要对应卸载，windows下直接下载发行版的msi后双击运行安装即可：安装地址，这里就不详细介绍了。

补充：NPM包的本地安装（推荐）和全局安装概念

NPM是Node附带的包管理工具，可以在NPM服务器下载或者上传包程序，我们用的HEXO用的就是NPM安装的。很多大神写的网页插件也是以Node包<bag>的形式发布的，这里简单介绍一下相关命令，后续在插件安装时会用到。

安装包：打开cmd或者Git bash。

  • 全局安装：-g为全局安装，全局安装在命令行中运行组件包命令。

  • 本地安装：去掉 -g，使用时直接在<bag>.config.js中require('gulp')

    这里的devDependencies是开发环境，在项目上线后就没有关系了，比如gulp、webpack等。

  • 全局安装可能需要解决包路径的配置问题且不好管理更新。尽量本地安装，能够保证不同版本包的依赖，加载速度会更快，但占的空间会相应变大。

Step 3：Git安装

  1. 官网下载：Git官网。

    • 默认安装选项即可。
    • 下载之后，在目标文件夹右键可打开Git Bash，git的相关命令以后就可以在这里运行。注意，安装过程可能会有一个默认”Use Git from the Windows Command Prompt“的选项，表示命令在window的cmd命令行中也可以运行。

    Git Bash 是一个shell，windows下的命令行工具，可以执行Linux命令，是基于cmd的。安装的时候会将cmd自动添加到环境变量中。

    Shell：与“核”进行人机交互的“壳”。简单理解为处理输入，调用内核接口的交互页面，一般为命令行形式。另外Shell也指一种命令语言或者程序设计语言，比如Windows Exploerer是图形界面Shell。

  2. 在命令行设置姓名和邮箱地址，这里的信息对应了电脑提交到github的署名。global参数表示所有的Git库上都会用这个参数，也可以局部配置。

  3. 建立本地与github账户的连接，使你的github账户允许接收你电脑的推送。

    • 首先打开电脑文件夹，找到C:\Users\Name\.ssh文件夹并删除，右键打开git bash生成公私钥:

    • 输入三次回车，打开.ssh文件夹，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，这就是你的公钥。

    • 打开github主页，点击右侧头像，依次点击setting -> access->ssh and gpg keys，新建SSHkey，把复制的内容粘贴进去，自己命名保存。表示你的github账号接受和这个公钥主体的推送。

    • cmd中输入ssh -T git@github.com验证，如果崩出来个选项输入yes回车，如果返回Hi！ <name>，You've...即成功。

  4. 查看你的git配置信息，另外可以添加GIT的全局配置，配置第三方编辑器等，这里不多介绍了。

  5. 检查环境变量

    会自动配置path环境变量<安装目录>\git\Git\cmd，可以查看

  6. github注册并新建仓库，命名为<user>.github.io，其他默认选项，新建空白文件index.html并提交

    这里命名一定要按照设置来，这也是公网访问你网页的域名

Step 6：备份

如果前面的步骤都完成了，那么恭喜，博客已经支持基本的文字和代码编辑，上传访问功能了。

但是有关图片、公式以及网站的图标等个性化设置还没有做，在做之前需要备份，否则如果崩了只能从头来过。

github寄存的是已经编译渲染过的网页代码，但是其他文件例如博文、配置文件并不会上传，所以这里我们新建一个仓库用于存储我们所有的源代码。

  1. 在GitHub中新建空仓库，例如命名为Hexo_Backup，权限为private，分支为main

  2. 根目录文件夹初始化git仓库，然后与远程仓库关联

    如果关联错了，git remote rm orgin解绑，这里的origin就是默认的远程库的名字，在此之前可以使用git remote -v查看远程库信息

    如果报错分支不存在的话，可能是本地的master分支和main无法对应，只需要修改本地分支名即可：

    或者还有一种办法是：直接将远程的空白仓库克隆到任意位置，然后复制里面的.git文件到根目录文件夹中，也能达到本地仓库与远程仓库的关联效果（好像更方便）。

  3. 为防止冲突，删除或者移动主题文件夹中的.git文件夹

    我个人在备份时发现不管是否删除，上传时都会跳过butterfly这个文件夹，后续备份的时候也会跳过，可能涉及到仓库的嵌套上传和权限问题。

    如果和我一样，那么后续在克隆备份的时候需要再克隆一遍butterfly的仓库，即再次进行step5。

  4. 创建或者修改.gitignore，用于忽略一些文件，比如配置环境时添加的一些内容

    需要替换的文件：config.yml，theme/，source/，scaffolds/，package.json，.gitignore

    之后更新就通过git同步到备份仓库中，如果哪天玩崩了再恢复前一个版本。

我一般使用备份的用途：

  • 双设备开发，偶尔回寝室只有笔记本的时候想写就将备份库的内容git pull到本地后开发，完成后再git push
  • 莫名其妙崩盘后可以通过本地reset回退到正确版本，实在不行就clone github上的稳定版本。

Step7：写作（可选）

博文是Markdown形式，所以这一步推荐编写的工具组typora+picgo，如果有自己的写作工具可以跳过。

  1. 选择MD编辑器：typora

    typora现在是付费版，也可以选择其他编辑器。

    picgo官网下载即可。

  2. 选择图床：将图片存在图床，不需要通过每次都上传图片，而是通过访问图床就能在网页中显示图片

    上传到博客仓库的图片一多很容易拖累网站，所以一般只用来保存头像啥的。

    • smms：直接官网注册获取token在picgo里设置即可，50G的容量够用，目前看来还算稳定
    • github+jsdeliver：可能有点限速，毕竟是通过公开库存图片
      • github新建仓库blog_img，权限设置为公开
      • 访问https://github.com/settings/tokens ，点击Generate new token，名称自定义，勾选repo然后生成token，需要及时复制（仅显示一次）
      • 打开picgo软件
        • 设定仓库名：<user>/blog_img
        • 设定分支名：master
        • 填写token
        • 自定义域名来加速：https://cdn.jsdelivr.net/gh/<user>/blog_img

    图床就是在薅羊毛，很多小网站可能不稳定，记得备份重要图片。

  3. 在Typora中设置图片上传选项，点击左上角：文件->偏好设置->图片，在上传服务中选择picgo，然后路径选择pigco.exe的路径，点击检验图片上传选项，成功即可。
