C++ 速成

概述

在实际使用过程中，c++能够兼容c并且添加独有功能，故一般都编写cpp文件了，然后即用即查。

面向对象的设计

  1. 封装：对外仅公开接口
  2. 继承：已有类派生新类，继承属性方法且可扩展或修改
  3. 多态：同种操作对不同对象有不同解释实现，通过接口或者继承实现
  4. 抽象：从具体实例中提取共同特征，形成抽象类或接口

这里的封装是指定义内部函数接口，只需要调用函数即可，而抽象只指内部私有数据仅对类的成员开放，也就是说内部细节隐藏。

标准库

  1. 核心语言
  2. C++ 标准库
  3. 标准模板库（STL）

静态变量和静态成员函数

静态变量

  • static 关键字来把类成员定义为静态的，论创建多少个类的对象，静态成员都只有一个副本，即在类的所有对象中是共享的。
  • 初始化为0
  • 可以在类的外部通过使用范围解析运算符 :: 来重新声明静态变量从而对它进行初始化

静态成员函数

  • 即使在类对象不存在的情况下也能被调用
  • 只要使用类名加范围解析运算符 :: 就可以访问
  • 只能访问static成员（静态数据、静态成员函数）和类外部的其他函数
  • 有一个类范围，不能访问类的 this 指针，可以用来判断某些对象是否被创建

派生类继承除外的内容：

  1. 内部成员函数中的三个特殊函数：构造、析构和拷贝构造函数。
  2. 基类的重载运算符
  3. 基类的友元函数。

继承类型

一个类派生自基类：常用public

  • 公有继承（public）：基类的公有和保护成员也是派生类的公有和保护成员，私有成员不能直接被派生类访问，但是可以通过调用基类的公有和保护成员来访问。
  • 保护继承（protected）：基类的公有和保护成员将成为派生类的保护成员。
  • 私有继承（private）：当一个类派生自私有基类时，基类的公有和保护成员将成为派生类的私有成员

多态

类之间存在层次结构，并且类之间是通过继承关联时，就会用到多态

调用成员函数时，会根据调用函数的对象的类型来执行不同的函数

def 解释
静态多态，或静态链接，早绑定 多态函数被设置为基类中的版本
动态链接，或后期绑定 在程序中任意点可以根据所调用的对象类型来选择调用的函数
虚函数 基类中使用关键字 virtual 声明的函数，不使用静态链接
纯虚函数 在基类中不能对虚函数给出有意义的实现

比如通过形状派生的三角和正方形类别中计算面积的函数不同，但逻辑意义相同：

  • 放置关键字 virtual，编译器看的是指针的内容，而不是它的类型
  • 虚函数的目的是允许使用基类指针调用子类该函数
  • 纯虚函数才代表函数没有实现，只是为了实现一个接口，规范继承这个类的人需要实现这个函数

数据类型 解释
